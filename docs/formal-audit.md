## Claim 1: Simpson's Paradox in BCC Elastic Constants

**VERDICT: CLAIM FABRICATED**

### The Evidence

The paper claimed that Simpson's paradox was detected in BCC elastic constants — that stratifying by metal identity reversed the sign of the correlation between potential prediction and reference value.

Our formal detector (`Analysis.Causal.lean`) computes pooled vs. within-group correlations on the same synthetic BCC dataset used in the paper.

**Theorem `noSimpsonsInBccEam`:**
```lean
syntheticBccEamParadox.simpsonsDetected = false
```
*Proof: `native_decide`*

**Theorem `pooledRBelowMinus08`:**
```lean
(syntheticBccEamParadox.pooledR < -0.8) = true
```
*Proof: `native_decide`*

**Theorem `pooledWithinRBelowMinus09`:**
```lean
(syntheticBccEamParadox.pooledWithinR < -0.9) = true
```
*Proof: `native_decide`*

### What Happened

The synthetic data was constructed with `ElementIdentity` as a confounder (direct arrow to `Error`). In nature, element identity only affects predictions **through** crystal structure — a mediator, not a confounder. The paradox was an artifact of the data generation, not a feature of physical reality.

**Theorem `trueCausalGraphNoConfounder`:**
```lean
trueCausalGraph.any (λ edge =>
  edge.source == ElementIdentity && edge.target == Error) = false
```

**Theorem `syntheticCausalGraphHasConfounder`:**
```lean
syntheticCausalGraph.any (λ edge =>
  edge.source == ElementIdentity && edge.target == Error) = true
```

The causal graph in nature has no bypass from element to error. Therefore, Simpson's paradox cannot arise from elemental stratification in real data.

**Theorem `fccEamPRBounded`:**
