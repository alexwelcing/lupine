# Lupine Mobile Perfection Strategy (Stitch Integration)
> **Goal:** Optimize mobile viewports for the existing `lupine-site` and `glimPSE` viewers within the current monorepo architecture, leveraging the new Stitch project (`8236093918010889121`) to generate high-fidelity, touch-native UI components.

## 1. Architectural Approach
We are **not** building a separate application or routing subdomain (like `m.lupine.science`). We will optimize the existing desktop codebase by integrating Stitch-generated mobile components using responsive techniques and dynamic rendering.

*   **Marketing Pages (`lupine-site`)**: Implement strict Tailwind CSS breakpoints (`md:`, `lg:`). Stitch will be prompted to generate `MOBILE` device screens, which we will extract and map to the `<div class="block md:hidden">` tree.
*   **Glim Viewer (`atlas-view`)**: Instead of CSS hiding, we will use React's `matchMedia` (e.g., `useMediaQuery`) to physically swap out Heavy Desktop Panels (Sidebars, floating stats) for Lightweight Mobile primitives (Bottom Sheets, Collapsible drawers, FABs) generated by Stitch.

## 2. Stitch Interaction Loop
Our workflow with the Stitch MCP will follow these steps:

1.  **Contextual Prompts**: I will instruct the `generate_screen_from_text` Stitch tool using the `deviceType: "MOBILE"` parameter.
2.  **Generative Targets**: 
    *   *Mobile Hero / Nav*: Generate a mobile glassmorphic hamburger menu and touch-friendly hero CTA.
    *   *Mobile Viewer HUD*: Generate a bottom-sheet based control interface for the `<Canvas>`, keeping the center of the screen empty for gestures.
    *   *Data Tables*: Generate mobile-optimized horizontal scrolling timeline/data cards to replace the massive desktop tables.
3.  **Extraction & Migration**: I will pull the generated React components or HTML from Stitch and seamlessly inject them into the existing files.

## 3. High-Priority Mobile Interventions

### A. The lupine-site Marketing Funnel
*   **Navigation & Header**: Replace the cramped top-nav with a Stitch-generated full-screen blur overlay menu.
*   **The Quantum Workflow Timeline**: The side-by-side desktop timeline will be restructured into a vertical, single-column scroll-snapped list. Touch targets for the CTA buttons will be expanded to a minimum of 48x48px.

### B. The glimPSE Viewer Array
*   **Touch Controls**: The `OrbitControls` inside the WebGL `<Canvas>` will be tuned for pinch-to-zoom and two-finger pan.
*   **Bottom Sheet HUD**: Desktop floating panels (Settings, Playback, Stats) overlap each other on mobile. We will generate a unified "Bottom Sheet" in Stitch that slides up and houses all toggles (Bloom, SSAO, Speed).

## 4. Execution Plan
1.  **Verification**: Confirm the Stitch project `8236093918010889121` is accessible to the MCP after the IDE reboot.
2.  **Generation Phase**: Rapidly prompt Stitch for the 3 core mobile views (Marketing Nav, Mobile Hero, Mobile HUD Bottom-Sheet).
3.  **Integration Phase**: Inject generated React logic into `@lupine/ui` components and the marketing HTML files, utilizing Tailwind media queries (`max-w-md`) to ensure total mobile perfection.
