Proving the Ribbon — What the Machine Said

The KIMI brief asked for six experiments. Four ran for real on a local RTX A4500 host; two are honestly blocked. No number on this page is invented — including the ones that contradict the predictions.

Host Python 3.14 · numpy 2.3 · RTX A4500 Framework lupine_framework (shipped, 10/10 tests) Data 38-potential / 10-element drop + 3 MLIPs
scroll ↓

Ran Experiments 1, 3, 5, 6 with the shipped lupine_framework against the real data drop. Exp 6 PASSES (incremental SVD <0.2% vs batch, sub-ms). Exp 3 PASSES (PR stays bounded ~1.2–1.3 as properties grow — does not inflate). Exp 5 is PARTIAL (the strong-alignment core Ag/Au/Pt/Ta transfers; Cr/Nb/Pb don't). Exp 1 produced a real surprise: on the shipped subset Au's local PR does not jump (Δ +0.005) — which actually confirms Part 2's thesis that Au's escape is a cross-paradigm displacement, not a local dimensionality inflation — while Ni is the only element that trips the ΔPR>0.5 rule. Exp 2 & 4 are BLOCKED: they need MACE/Orb/ASE + a DFT binary; the GPU is here, the MLIP/DFT software stack is not. I also caught and fixed a bug in my own Exp-6 metric before reporting it.

This is the third page of the series — Part 1 (framework), Part 2 (validation), and this compute log. The runner and raw results are shipped for reproducibility: run_experiments.py ↗ · experiment_results.json ↗.

⚠ Honest scope The hand-off contained a 38-potential / 10-element classical subset (Ag Al Au Cr Cu Fe Mo Ni V W), not the full 559-potential corpus (server-side). Per-element PR is only computed where a element had ≥4 classical potentials (7 of 15 qualify). Every cell below states its n. The methodology is exact; the coverage is what shipped.

The framework's IncrementalSVD (Brand 2002) tracked the top-5 covariance eigenvalues to within 0.18% of a full batch SVD over 10,000 correlated 6-D observations, at sub-millisecond median latency, numerically stable throughout.

⚠ Self-caught measurement bug My first run reported "rel error = 1.0" and I nearly flagged a framework defect. On inspection that was my harness's fault: rank=5 intentionally truncates the 6th component to 0, and I was comparing that deliberately-zeroed tail against batch's non-zero λ₆ (|0−0.035|/0.035 = 1.0). Comparing only the retained rank gives 0.0018. Reported here because honest negative-then-corrected results are the point.
Verdict — PASS. Both framework claims hold on this host: <1% vs batch (actual 0.18% on retained rank) and sub-ms per update (median 0.09 ms; lone 0.47 ms first-call warmup). Suitable for mid-run MD filtering as designed.

Exp 2 & 4Blocked — and why

Orb-v3 relativistic toggle · Au nanocluster generation

BLOCKED

These require an MLIP inference runtime (Orb-v3 / MACE), ase, and a DFT binary (VASP / Quantum ESPRESSO with SOC). The host has the GPU (RTX A4500, CUDA-capable) but none of that software stack is installed, and standing up VASP/QE + foundation-MLIP weights is out of scope for a local run — so these are reported blocked rather than fabricated.

  • Exp 2 surrogate (data-only, already in Part 2): across the three shipped MLIPs Au's error direction is the most self-consistent of all 15 elements (mean cosine 0.984) — the relativistic mis-handling is reproducible, which is the precondition the rel-toggle experiment would exploit. The toggle itself needs an Orb-v3 runtime with the relativistic flag.
  • Exp 4: needs ASE + MACE(CUDA) geometry optimisation plus DFT single-points; no partial surrogate is honest here.
To unblock: a host (or the 20 GB vGPU node) with mace-torch/orb-models + ase and a DFT binary. The framework hooks (Regulator.check mid-optimisation) are ready; only the simulation backend is missing.

BackIntegration requests & what's next

From running it, the concrete asks back to the framework team:

  • Cross-paradigm escape statistic. Exp 1 shows local ΔPR is the wrong instrument — add a first-class detect_paradigm_shift(element) that measures classical-centroid vs MLIP-centroid separation (direction + magnitude), not just cloud dimensionality.
  • Ni follow-up. Ni is the only ΔPR>0.5 element on the subset (1.42→2.34). Either a genuine second escape or a small-n artifact — needs the full 559 corpus to disambiguate.
  • Ship the full corpus. 8/15 elements were unscoreable here for lack of classical potentials. The methodology is proven; it needs the 559-potential matrix to produce the publishable 15×5 table.
  • get_eigenvalues() contract. Document that it returns the rank-truncated spectrum (tail = 0) so downstream error metrics compare on the retained rank — would have prevented my harness bug.

Everything reproducible: run_experiments.py ↗, experiment_results.json ↗. The ribbon thesis stands at the global level (bounded PR across the paradigm jump, bounded under property growth); the per-element escape definition needs sharpening to cross-paradigm — which the framework can absorb.
