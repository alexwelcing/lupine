Growing the Hyper-Ribbon

A mathematical framework for conditional manifold extension, real-time escape detection, and the Lupine regulator.

Lupine Materials Science · Seattle, WA 559 classical potentials · 15 metals · 3 foundation MLIPs Status mathematical scaffold
scroll ↓

The Lupine hyper-ribbon is a low-dimensional error manifold (participation ratio PR ≈ 1.05–1.86 in 3D property space) discovered across 559 classical interatomic potentials and 15 benchmark metals. When foundation MLIPs (MACE-MP-0, CHGNet, Orb-v3) are added, 14/15 elements stay on the ribbon — but gold (Au) escapes, its PR jumping from 1.02 → 1.67 under MACE. This document develops the machinery to (1) grow the manifold to arbitrary N+ properties while keeping PR bounded via conditional activation, (2) detect escape events in real time using a Participation-Ratio statistic, (3) build a regulator acting as a configurable mid-run filter (an audio-EQ analogy), and (4) propagate escape discoveries through a catalog that automatically improves all future MLIP runs. Gold nanostructure generation is the validation target — each Au structure feeds back into the catalog, tightening the ribbon for everyone downstream.

Section 1The Core Geometry: Hyper-Ribbon as Error Manifold

1.1Sloppy Models and Information Geometry

The mathematical foundation of the Lupine platform rests on the sloppy model formalism developed by Transtrum, Machta, and Sethna 1912. In their 2011 Physical Review E paper, they proved that multiparameter models across physics, biology, and materials science exhibit a universal geometric structure: the model manifold — the surface traced by model predictions as parameters vary — is not a generic high-dimensional surface but a hyper-ribbon, a manifold whose widths decay geometrically across many orders of magnitude. The Fisher Information Matrix (FIM) serves as the natural Riemannian metric on this manifold, and its eigenvalue spectrum encodes the hierarchy of parameter sensitivities.

The FIM for a model with parameters θ predicting observables y is:

g_{\mu\nu} = \left\langle \frac{\partial^2 \log P}{\partial \theta_\mu \partial \theta_\nu} \right\rangle

For the interatomic potential problem, we do not work in parameter space but in prediction-error space: each potential φ ∈ ℳ (the manifold of published potentials) produces a vector of prediction errors ε(φ) = (ε₁(φ), …, εₙ(φ)) where each component is the signed error for a specific material-property pair. The error manifold ℳerror ⊂ ℝⁿ is the set of all such error vectors. The sloppy-model claim, translated to this setting, is that ℳerror occupies a hyper-ribbon — a low-dimensional ridge within the ambient n-dimensional space.

Machta et al. (2013) 69 connected this parameter-space compression to the emergence of effective theories in physics. Their key insight: the same eigenvalue hierarchy that makes continuum mechanics emergent from quantum field theory also makes interatomic potential errors predictable and correctable. The Lupine program operationalizes this insight by treating the ribbon as a corrective substrate rather than merely a descriptive curiosity.

1.2Participation Ratio as Dimensionality Measure

The Participation Ratio (PR) provides a theoretically grounded measure of effective dimensionality that is superior to threshold-based eigenvalue counting 1314. For a covariance matrix Σ with eigenvalues λᵢ, the PR is:

D_{PR} = \frac{\operatorname{Tr}(\Sigma)^2}{\operatorname{Tr}(\Sigma^2)} = \frac{\left(\sum_{j=1}^N \lambda_j\right)^2}{\sum_{j=1}^N \lambda_j^2}

The PR quantifies how many eigenmodes are necessary to "significantly capture" the distribution of data 13. It interpolates between correlation dimension at small scales and global dimension at large scales, and it has been independently discovered in atomic spectroscopy (the original participation ratio), condensed matter physics (a localization measure), quantum information (purity), economics (the Herfindahl-Hirschman Index), and machine learning (kernel expressivity).

For the IMMI benchmark, the global PR of 1.05–1.86 (of 3 nominal dimensions for C₁₁, C₁₂, C₄₄) means that the error vectors, despite living in ℝ³, effectively occupy a 1.05–1.86 dimensional manifold — consistent with a 1D curve or a highly anisotropic 2D ribbon. This is the empirical signature of sloppy-model universality applied to interatomic potentials.

This specificity is what makes the escape catalogable and correctable.

Section 2Growing the Manifold: N+ Properties with Conditional Activation

2.1The Extension Problem

The current Lupine manifold spans three elastic constants (C₁₁, C₁₂, C₄₄). The de-myopization result 6 showed that adding the lattice constant a₀ — forcing a joint Cᵢⱼ + a₀ manifold — preserves the ribbon structure (PR 1.05–2.05). The natural next step is extending to four property families (adding Ecoh, B₀ via the Phase-D compute lane) and ultimately to arbitrary N properties. The critical question: does the ribbon survive as N grows?

Mathematically, we are asking whether adding new columns to the error matrix E (potentials × properties) preserves the low-rank structure. The answer depends on the correlation structure of the new properties with existing ones. If new properties are largely predictable from existing error modes — if they lie approximately in the span of the current ribbon — the PR grows only logarithmically. If they introduce genuinely new orthogonal error structures, the PR grows linearly, and the ribbon thesis breaks.

2.2Conditional Activation: Sparse Property Inclusion

The key insight for handling N+ properties is that not all properties are relevant for all contexts. The lattice constant a₀ matters more for surface calculations than bulk; cohesive energy Ecoh matters for nanoparticle stability; magnetic moments matter only for Fe, Co, Ni, Mn. Rather than building a single dense N-dimensional manifold, we propose a conditionally activated sparse manifold.

For a given simulation context c = (element, structure, temperature, task, …), only a subset of properties P(c) ⊆ {1, …, N} is active. The effective manifold dimensionality is measured on the active subspace ℝ|P(c)|, not the full ambient ℝN. This is analogous to sparse PCA 3843, where principal components are constrained to use only a subset of features — but here the sparsity pattern is determined by physical context rather than statistical optimization.

Ambient space:      ℝ^N   (all registered properties)
Active selector:    σ_c: {1,...,N} → {0,1}   (context-dependent mask)
Active subspace:    ℝ^n  where n = |{i : σ_c(i) = 1}|
Projected manifold: M_c = Π_c(M) ⊂ ℝ^n   (ribbon on active props)
Effective PR:       PR(M_c) = (Σ λ_i)² / Σ λ_i²   (on active subspace)

The GrowingManifold class implements this structure. Properties are registered with activation_conditions — predicates that evaluate true when the property should be included. Core properties (C₁₁, C₁₂, C₄₄) are always active; conditional properties (a₀, Ecoh, μmag, γsurf, rel_corr) activate only when their conditions are met.

2.3Why PR Stays Bounded

The PR grows sublinearly with N because new properties in materials science are not statistically independent. The elastic constants C₁₁, C₁₂, C₄₄ are coupled through the bulk modulus B = (C₁₁ + 2C₁₂)/3 and shear modulus G = (C₁₁ − C₁₂)/2. Adding B or G as explicit properties would not introduce new orthogonal error modes — they are linear combinations of existing ones. Similarly, a₀ correlates with C₁₁ through the potential's equilibrium condition. The de-myopization result empirically confirmed this: adding a₀ only increased the PR from ~1.5 to ~1.7, not to 2.0+ as an independent dimension would.

The mathematical bound comes from the covariance structure of materials properties. For a covariance matrix Σ, PR is maximized when all eigenvalues are equal (PR = N, isotropic) and minimized when one eigenvalue dominates (PR → 1, a 1D line). Materials properties exhibit strong correlations because they derive from the same underlying interatomic potential energy surface. PR therefore stays in the 1–3 range even as N grows to 10+, provided new properties come from physically meaningful families rather than arbitrary combinations.

Escape events like Au represent genuine new dimensions — error structures orthogonal to the ribbon that cannot be predicted from existing modes. When rel_corr is activated for Au, the PR may spike (as observed: 1.02 → 1.67), signaling that relativistic corrections introduce a new error mode. The regulator's job is to detect this spike and apply the appropriate correction.

3.3Mathematical Formalization

Let x(t) ∈ ℝⁿ be the MLIP's predicted property vector at MD step t, and xref the reference (DFT or experimental) value. The error vector is ε(t) = x(t) − xref. The regulator operates through three operations.

Step 1 — Ribbon Projection. Project ε onto the ribbon subspace spanned by the top-k eigenvectors {v₁, …, vₖ} of the error covariance matrix:

\varepsilon_{\text{ribbon}} = \sum_{i=1}^{k} (\varepsilon \cdot v_i)\, v_i

Step 2 — Escape Detection. Measure the escape ratio:

\rho_{\text{escape}} = \frac{\lVert\varepsilon\rVert^2 - \lVert\varepsilon_{\text{ribbon}}\rVert^2}{\lVert\varepsilon\rVert^2} = \frac{\lVert\varepsilon_{\perp}\rVert^2}{\lVert\varepsilon\rVert^2}

If ρescape ≥ εthreshold, an escape is detected.

Step 3 — Correction. Apply the element-specific correction from the catalog. For Au this is a relativistic overlay Δrel shifting the d-electron energy levels:

\varepsilon_{\text{corrected}} = \varepsilon - \Delta_{\text{rel}}(\text{element},\, \text{MLIP})

The correction Δrel is learned from the escape catalog: the mean error vector for Au on the given MLIP, projected onto the escape subspace. This is analogous to the Kalman gain 4955 — an optimal linear estimator that minimizes the mean-square error of the corrected prediction.

Section 4Escape Detection and the Catalog Propagation System

4.1The Escape Catalog

The escape catalog is the accumulated knowledge base of all escape events detected across all MLIPs, elements, and contexts. Each entry records element, MLIP, escape ratio, local PR, context (structure/temperature/task), correction applied, correction success, and timestamp.

The catalog is append-only and versioned. Each new entry triggers a propagation event: all users running simulations involving the catalogued element on any MLIP receive the correction automatically. This is the network effect of the platform — each escape discovered improves accuracy for all downstream users, regardless of which MLIP they use.

4.2Detection Algorithm

Algorithm: DetectEscape(element, mlip, context)
   1. P_active ← GetActiveProperties(context)
   2. n ← |P_active|
   3. if n < 2: return INSUFFICIENT_DATA
   4.
   5. // Build error matrix for this context
   6. E ← []                       // list of error vectors
   7. for each potential_id in error_data:
   8.   if context_matches(potential_id, context):
   9.     E.append(error_data[potential_id][P_active])
  10.
  11. if |E| < 3: return INSUFFICIENT_DATA
  12.
  13. // Covariance + eigendecomposition
  14. Σ ← Covariance(E)
  15. λ, V ← Eigendecomposition(Σ)   // λ sorted descending
  16.
  17. // Ribbon dimension (90% variance)
  18. k ← min{ i : cumsum(λ)/sum(λ) ≥ 0.9 }
  19.
  20. // Element-specific escape ratio
  21. ε_element ← mean error for element in E
  22. coeffs   ← Vᵀ · ε_element
  23. E_ribbon ← sum(coeffs[1:k]²)
  24. E_escape ← sum(coeffs[k+1:n]²)
  25. ρ_escape ← E_escape / (E_ribbon + E_escape)
  26.
  27. PR_local ← (sum(coeffs²))² / sum(coeffs⁴)
  28.
  29. if ρ_escape ≥ threshold:
  30.   RecordEscape(catalog, element, mlip, ρ_escape, PR_local, context)
  31.   return ESCAPE(ρ_escape, PR_local, suggested_correction)
  32. else:
  33.   return ON_RIBBON(ρ_escape, PR_local)

The algorithm is O(m · n²) where m is the number of error observations and n the active subspace dimension. With n typically 3–8 and m in the hundreds, this is computationally negligible compared to the MLIP evaluation cost.

Section 5Gold Nanostructure Generation: A Validation Target

5.1Why Gold?

Gold is a uniquely valuable validation case because its relativistic effects 606466 create a genuine physical mechanism for escape that is simultaneously well-understood theoretically and challenging for MLIPs to capture. The key effect is s-orbital contraction: the 6s electrons move at ~0.6c, increasing their effective mass by ~20% and pulling the orbital inward. By orthogonality, the 5d orbitals expand outward and are destabilized. The 5d → 6s interband transition shifts from the ultraviolet (as in silver) into the visible blue range — which is why gold appears yellow 64.

For MLIPs, the d-electron treatment must account for relativistic scalar corrections that shift the relative energies of s and d orbitals. Foundation MLIPs trained on PBE-DFT data without explicit relativistic corrections in their feature representations will systematically misrepresent Au's electronic structure, producing errors that are orthogonal to the ribbon (escapes) because the ribbon was constructed from non-relativistic or weakly relativistic elements.

The Lupine contribution is not a new generation algorithm but a corrective layer that improves whatever MLIP the user chooses.

5.3Feedback Loop

Each gold structure generated produces a feedback event: (1) the MLIP predicts properties for the Au structure; (2) the regulator checks whether the error vector stays on the ribbon; (3) DFT validation provides the ground-truth error; (4) if Au escapes, the catalog is updated with the new context (nanoparticle, size, ligand environment); (5) the updated catalog propagates to all future Au simulations across all MLIPs.

This creates a virtuous cycle: more Au generations → richer escape catalog → better corrections → more accurate predictions → more reliable generations. The target is reducing Au's PR from 1.67 → < 1.2, bringing it back onto the ribbon and making Au as well-behaved as Ag or Pt.

Section 6Formal Structure: Toward a Lean Specification

6.1Type Signature

The framework can be formalized in a theorem-proving language like Lean 35, following the precedent of formalized chemical physics proofs. The core types:

-- A property is a named, categorized observable
structure Property where
  name          : String
  category      : PropertyCategory
  index         : Nat
  always_active : Bool

-- Context determines which properties are active
structure Context where
  element        : String
  structure_type : String
  temperature    : Float
  task           : String

-- A property is active in a context if its conditions hold
def is_active (p : Property) (c : Context) : Prop :=
  p.always_active ∨ ∃ cond ∈ p.activation_conditions, cond c

-- The error manifold for a given context
def ErrorManifold (N : Nat) (properties : Fin N → Property) (ctx : Context) :=
  { ε : Vector Float N // ∀ i, (is_active (properties i) ctx) ∨ (ε i = 0) }

-- Participation ratio measures effective dimensionality
def participation_ratio {n : Nat} (eigenvalues : Vector Float n) : Float :=
  (eigenvalues.sum ^ 2) / (eigenvalues.map (· ^ 2)).sum

6.2Key Invariants

Invariant 1 (Ribbon Boundedness). For any context c, the participation ratio of the active error manifold is bounded: PR(Mc) ≤ PRmax where PRmax ≈ 3.0 is a platform constant. This invariant is violated during escape events — which is exactly how escapes are detected.

Invariant 2 (Escape Specificity). If element e escapes on MLIP m, then for any other element e′ ≠ e, either e′ also escapes (a class effect) or e′ stays on the ribbon (an element-specific effect). The Au/Ag distinction proves Au's escape is element-specific.

Invariant 3 (Correction Monotonicity). Applying a catalogued correction to an escape event never increases the participation ratio: PR(εcorrected) ≤ PR(εraw). This ensures corrections genuinely improve predictions rather than introducing new errors.

6.3The Core Theorem

Theorem (Hyper-Ribbon Correction). Let ℳ be the manifold of interatomic potentials, E: ℳ → ℝN the error map, and R ⊂ ℝN the hyper-ribbon subspace. For any potential φ ∈ ℳ and context c, if E(φ) projected onto the active subspace has escape ratio ρ < εthreshold, then there exists a correction Δc(element, MLIP) such that the corrected error E(φ) − Δc lies in R.

The theorem is empirically grounded rather than analytically proven — it holds for the 559 classical potentials and 3 foundation MLIPs tested to date, with Au as the primary confirmed exception that motivates the correction machinery. The platform's value proposition is that this empirical regularity, combined with systematic escape detection and cataloging, provides a practical corrective substrate even in the absence of a full analytical proof.

7.2Incremental Updates

The decomposition engine uses incremental SVD 4852 to update the eigenbasis as new error data arrives without recomputing from scratch. For a new observation xn+1: (1) decompose the centered vector x̃n+1 = Un cn+1 + x̃n+1⊥; (2) update the covariance approximation Δn+1 = (n/(n+1)) Δn + (n/(n+1)²) x̃n+1 x̃n+1T; (3) compute SVD of the updated system, keeping only the top-q components. This is O(q²) per update (q = 3–5 typically), suitable for real-time operation during MD.
